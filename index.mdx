Terraform AWS MEAN Stack is an Infrastructure-as-Code project that provisions a production-ready, multi-tier web application environment on AWS. It orchestrates six reusable Terraform modules to stand up a complete networking layer, load balancing, two Node.js application servers, and a private MongoDB database — fully automated from a single `terraform apply`.

## What Gets Deployed

A single `terraform apply` provisions the following AWS resources:

<CardGroup cols={3}>
  <Card title="Networking" icon="network-wired">
    1 VPC, 2 public subnets, 1 private subnet, Internet Gateway, NAT Gateway, and route tables.
  </Card>
  <Card title="Compute" icon="server">
    3 EC2 instances — 2 Node.js/Nginx app servers in public subnets and 1 MongoDB server in the private subnet.
  </Card>
  <Card title="Load Balancing" icon="scale-balanced">
    An Application Load Balancer distributing HTTP traffic across both Node.js instances with health checks.
  </Card>
  <Card title="Security" icon="shield-halved">
    3 layered security groups controlling ALB, Node, and MongoDB access, plus IMDSv2 enforcement.
  </Card>
  <Card title="IAM" icon="key">
    An EC2 IAM role with the AmazonSSMManagedInstanceCore policy for session manager access without bastion hosts.
  </Card>
  <Card title="Remote State" icon="database">
    S3-backed Terraform remote state for safe, team-friendly infrastructure lifecycle management.
  </Card>
</CardGroup>

## Get Started

<Steps>
  <Step title="Install prerequisites">
    Install [Terraform >= 1.12](https://developer.hashicorp.com/terraform/downloads) and the [AWS CLI](https://docs.aws.amazon.com/cli/latest/userguide/install-cliv2.html), then configure your AWS credentials.
  </Step>
  <Step title="Create your S3 backend bucket">
    Create an S3 bucket in your target region to store Terraform remote state, then create a `backend.hcl` file with its details.
  </Step>
  <Step title="Configure variables">
    Copy `terraform.tfvars.example` to `terraform.tfvars` and fill in your project name, region, CIDR blocks, and allowed SSH IP.
  </Step>
  <Step title="Deploy">
    Run `terraform init -backend-config=backend.hcl`, then `terraform apply` to provision the full stack. Check the [Quickstart](/quickstart) for the complete walkthrough.
  </Step>
</Steps>

<Tip>
  After deployment, Terraform outputs the ALB DNS name, public IPs for both Node.js servers, the MongoDB private IP, and the path to your generated SSH key. See [Outputs](/deployment/outputs) for details.
</Tip>
